# Apply Checklist

## Preview Checklist

Before any mutation, confirm the preview bundle includes:

- exact target scope
- chosen mode
- pages to create
- pages to rewrite or repurpose
- pages to archive
- pages to keep live
- reading path changes
- owner boundary changes

## Apply Checklist

- Re-read the target subtree immediately before writing.
- Preserve existing useful content where possible.
- Archive stale or superseded pages instead of deleting by default.
- Keep root page human-readable and unnumbered.
- Keep child hub page names aligned to the numbered scaffold.
- Ensure each live truth has one owner page.

## Post-Write Verification Checklist

After applying, re-read:

- root page
- each live top-level hub page
- every page that became a new owner page
- each page that was archived or repurposed

Verify:

- metadata block exists on live pages
- Outline page body does not repeat the document title as an identical `H1`
- default reading path does not route through archive
- archive pages clearly look historical
- links point to live Outline pages
- no duplicate owner pages remain
- `24 Operational — User Guide` naming stays semantic if that branch exists

## Link Hygiene

- Use live Outline URLs for live pages.
- Do not leave stale placeholder links in the new structure.
- If a page was replaced, add a pointer note when needed.
- Avoid linking readers from live root pages straight into archive unless the task is provenance.
