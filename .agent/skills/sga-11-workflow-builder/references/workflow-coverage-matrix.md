# Workflow Coverage Matrix — Reference Guide

## Purpose
Template và hướng dẫn Phase A (Analyze) của workflow-builder skill.

## Coverage Matrix Template

```markdown
# Workflow Coverage Matrix — [Workspace Name]

## A-ESOAR → Workflow Mapping

| # | Value Chain Step | A-ESOAR Label | Workflow? | Workflow Name | Agent Owner | Skills Required |
|---|---|---|---|---|---|---|
| 1 | [Step name] | R (Robotize) | ✅ | `/[workflow]` | [Agent] | `[skill-id]` |
| 2 | [Step name] | A (Augment) | ✅ | `/[workflow]` | [Agent] | `[skill-id]` |
| 3 | [Step name] | S (Standardize) | ❌ N/A | — | Human SOP | — |
| 4 | [Step name] | R (Robotize) | 🔴 MISSING | — | — | — |

## Gap Summary

- Total R-steps: [N]
- Covered by workflow: [M]
- GAPS (R-step without workflow): [N-M] ← MUST be 0 before Gate 4
```

## Classification Rules

| A-ESOAR Label | Needs Workflow? | Rationale |
|---|---|---|
| **R** (Robotize) | ✅ MANDATORY | AI does 100%, must have deterministic process |
| **A** (Augment) | ✅ RECOMMENDED | AI assists 80%, needs structured handoff |
| **O** (Optimize) | 🟡 OPTIONAL | Human does it, workflow may help with tooling |
| **S** (Standardize) | ❌ NO | Human SOP, not agent workflow |
| **E** (Eliminate) | ❌ NO | Step removed entirely |

## Workflow Classification

### Tier 1: Infrastructure (Mandatory for Operations/BA)
- `start-session.md` — Morning briefing, crash recovery, memory load
- `end-session.md` — DIKW compression, session report, ledger update
- `checkpoint-session.md` — Mid-session state save

### Tier 2: Core Domain (From R-steps)
- Named after business process: `/build-report`, `/analyze-data`, `/review-submission`
- Each maps to exactly 1 R-step (or 1 A-step cluster)

### Tier 3: Support
- `onboard-new-user.md` — Generated by S05 (Training Designer)
- NOT built during Phase 3 Build

## Context Budget Rules

| Metric | Limit | Rationale |
|---|---|---|
| Max workflow length | 12,000 chars | Prevent context overload when Agent loads workflow |
| Max steps per workflow | 10 | Beyond 10 = split into sub-workflows |
| Max skills per workflow | 4 | Beyond 4 = workflow is doing too much |
